Белые Списки: Теория и Обход

Белые Списки(White Lists) это механизм с помощью которого разрешается доступ только к определённым ресурсам
Чем они отличаются от Чёрных списков?

Посмотреть различия можно в таблице ниже

Характеристика Белый список (Whitelist) Черный список (Blacklist)
Основной принцип Запрещено всё, что не разрешено явно. Разрешено всё, что не запрещено явно.
Логика проверки Система ищет объект в списке разрешенных. Система ищет объект в списке запрещенных.
Действие по умолчанию Блокировка (Default Deny). Пропуск (Default Allow).
Если объекта нет в списке Доступ отклоняется. Доступ предоставляется.

3. Добавление подписок

Throne: Приложение > Добавить профиль из буфера обмена > Создать новую группу подписки
Karing: Добавить профиль > Импорт из буфера обмена > Примечание можно указать любое > Под кнопкой подключения выбираем какой нибудь сервер
v2rayN: Серверы > Импорт массива URL из буфера обмена > Переходим в подписку > Группа подписки > Обновить подписку без прокси
NekoBox: Три полоски > Группы > Плюсик > Имя любое > Тип группы: Подписка > Ссылка для подписки вставляем ссылку

4. Глоссарий и тонкая настройка:

DoH (DNS over HTTPS) / DoT (DNS over TLS): Технологии шифрования DNS-запросов, предотвращающие слежку провайдера за тем, какие домены вы посещаете.

Stack (TUN): Программная реализация сетевого интерфейса. gVisor — это современный стек от Google, обеспечивающий высокую стабильность и безопасность.

MTU (Maximum Transmission Unit): Максимальный размер пакета данных. Значение 1500 критически важно для корректной работы протокола Hysteria2.

Рекомендуемые параметры:

  • Удаленный DNS: https://1.1.1.1/dns-query
  • Прямой DNS: 1.1.1.1
  • Stack (TUN): gVisor
  • MTU: 1500
  • Автообновление: каждые 2 часа
